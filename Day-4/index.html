3. What is callback hell?

Ans. Call back hell is a nested callback stacked below one another, in other words it is a function that will execute after execution of its parent function is executed completely. It also form pyramid like structure that is difficult to understand and debug. We should avoid using callback hell as much as possible. 


      function printNum(){
        setTimeout(() => {
          console.log('5')
          setTimeout(() => {
            console.log('4')
            setTimeout(() => {
              console.log('3')
              setTimeout(() => {
                console.log('2')
                setTimeout(() => {
                  console.log('1')
                },1000)
              },2000)
            },3000)
          }, 4000)
        }, 5000);
      }
      printNum()

Output :

      5
      4
      3
      2
      1

4. What are promises and why do we need them?

Ans. Promises is like heaven for callbacks. It is introduced in ES6 version to handle a asynchronous operations in an efficient way. It debug and manage code we need promises. It is basically used to find out if the asynchronous operation is successfully completed or not. A promise may have one of the three states that is pending, fulfilled or rejected. Promises allows us to write cleaner code and allows error to pass down the chain and can be handled in a common place. 


      var promise = new Promise((resolve, reject) => {
        if(5 === 5) 
          resolve();
        else 
          reject();
      });
         
      promise.
          then(function () {
              console.log('Success,  Both are equals');
          }).
          catch(function () {
              console.log('Some error, both are not equals');
          });

Output :

      Success,  Both are equals

5. What is promise chaining?

Ans. Promise chaining are easy to manage when dealing with multiple asynchronous operation where callbacks can create callback hell leading to unmageable code. To avoid callback hell we can use promise chaining. It is basically when multiple use of .then method are used. 


      const printNum = (time, value) => {
        return new Promise ((resolved, rejected) => {
          setTimeout(() => {
            console.log(time)
            resolved();
          }, time);
        })
      }
      
      printNum(5000, '5')
         .then(() => printNum(4000, '4'))
         .then(() => printNum(3000, '3'))
         .then(() => printNum(2000, '2'))
         .then(() => printNum(1000, '1'))

Output :

      5000
      4000
      3000
      2000
      1000
