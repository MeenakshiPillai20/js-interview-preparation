If write one line of code or 1000 line of code execution context always is created. Execution context have two parts that is creation phase and execution phase. In creation phase we have code environment or component and memory component Inside creation phase we have memory component and inside execution phase we have code component. Whenever we run a code a global execution context is being created in the call stack. Call stack is also called control stack and runtime stack. So in creation phase that is the memory component, all the variables and functions inside the code are assigned some memory that is undefined. Then inside the execution phase that is the code component if there is any variables defined with some values will be assigned to the variable and if there is any function call then a new function execution context is been created inside the global execution context inside the call stack. Whenever a function is called inside the code a new function execution context is been created all the time and it would be removed on the basis of last in first out. If in the new function execution context there is some variable which is not defined then the compiler start searching it inside the global execution context or the parent execution context. This way inside call stack after the global one by one function execution context created and the code inside it get executed after that it is it poped out in LIFO basis and finally global execution context also gets removed at Las and finally we get our output.

6. Explain passed by value and passed by reference?

Ans. In Pass by Value, function is called by directly passing the value of the variable as an argument. Basically the parameter passed as an argument creates its own copy. So any changes made inside the function is made to the copied value not to the original value.

In Pass by Reference function is called by directly passing the reference or address of the variable as an argument. Basically parameter passed as an argument does not create its own copy, it refer to an original value so changes made inside function affect the original value. In JavaScript array and object follows Pass by Reference property.
